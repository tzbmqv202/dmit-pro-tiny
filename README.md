## Pro Tiny 的线路到底怎么样

这是最核心的问题。

CN2 GIA 是电信运营商 AS4809 的骨干线路，区别于普通 CN2（AS4134）的地方在于：去程和回程都走 GIA 专用通道，不跟普通用户流量共享，拥塞程度低很多。晚高峰（20:00–23:00）是 VPS 线路最容易出问题的时段，CN2 GIA 在这段时间的表现普遍比 9929 和普通 CMI 稳定。

用下来的直观感受：晚高峰期间连接没有明显掉包，延迟比较稳定，不会出现那种一到晚上就卡成 PPT 的情况。

不过有几点要说清楚：

1. Pro 系列的 CN2 GIA 线路是有保证的，区别于 EB 系列（EB 系列走 CMIN2/9929，没有 GIA 保证）
2. 洛杉矶 CN2 GIA 延迟比香港和东京要高，大概在 150–180ms 左右，不是追求极低延迟的首选
3. 流媒体解锁方面，DMIT 用的是美国原生 IP，实测可以解锁 Netflix 等平台，但这不是官方服务保证，具体以当时实际结果为准

## Pro Tiny 适合谁，不适合谁

**适合的场景：**

- 搭建轻量代理节点，用于个人科学上网
- 个人博客或轻流量建站，日均访问量几千以内
- 跑定时任务、小型爬虫、自动化脚本
- 做开发测试环境，需要美国原生 IP
- 预算有限但对国内访问质量有要求的用户，$9.99/月算是 CN2 GIA 里比较低的门槛

**不太适合的场景：**

- 跑 MySQL、PostgreSQL 等数据库密集型应用（2GB 内存压力会比较大）
- 高流量的商业建站（1000GB/月对于有一定规模的站来说不够用）
- 需要极低延迟的应用（洛杉矶到国内 150ms+，香港或东京延迟更低）
- 跑 Java 应用或 JVM 系项目（内存消耗大，1核2G撑不住）

说句实话，Pro Tiny 定位就是"入门但不将就线路"。花 $9.99，你买到的不是高配机器，是一条靠谱的网络通道。这个逻辑搞清楚，就好做决定了。

## 购买注意事项和使用细节

几个容易踩坑的点：

1. **注册时关掉代理**。DMIT 会检测注册 IP，用代理注册容易被判断为欺诈然后自动退款（砍单）。直接用本机 IP 注册就行。

2. **登录用 SSH 密钥**。DMIT 默认不开密码登录，需要在购买时或控制面板里配置 SSH 公钥。不熟悉的话官网有教程。

3. **支付宝付款比 PayPal 便宜一点**。汇率问题，选支付宝会省几块钱人民币，尤其是年付的时候差异更明显。

4. **IP 被墙处理**。如果 IPv4 被墙（国内无法访问），可以在控制面板提交更换 IP 申请，每 15 天内一次免费，超出的话 $5 一次。

5. **退款政策**。3 天内使用流量不超过 30GB 的可以全额退款，超过 3 天在 30 天内则按剩余价值退。续费订单和同系列已退款 3 次的不在退款范围。

**Q：Pro Tiny 的流量 1000GB 是双向计算还是单向？**

A：双向流量（上行+下行合计）。日常代理和轻量建站一般用不完，如果是跑大量数据传输的业务就需要计算清楚。
